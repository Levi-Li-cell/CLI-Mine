# 项目目标（重规划版）

构建一套可长期运行（7x24）的 Claude-like AI 开发 Agent 系统：

- 前台：类似 Claude 的对话与工具协作体验
- 后台：具备无限上下文窗口下的持续开发能力（多轮 session 不丢状态）
- 治理：具备安全对齐、可观测、可回滚、可评估的工程化能力

## 设计原则（来自重点资料）

1. Anthropic 长时运行 Harness：
   - 初始化代理与编码代理分离
   - 持久化 handoff 工件（`feature_list.json`、`claude-progress.txt`、git 历史）
   - 每轮只完成一个特性，并做端到端验证后再标记完成

2. ReAct（Reason + Act）：
   - 统一采用“思考 -> 工具调用 -> 观察 -> 决策”的循环
   - 对外部环境用工具读取事实，降低幻觉

3. Function Calling / Tool Use：
   - 工具 schema 严格化（强约束参数、减少歧义）
   - 限制并发工具与可用工具集，优先确定性和可调试性

4. Constitutional AI：
   - 在系统层引入可执行的“行为宪法”
   - 对高风险请求默认转入解释/拒绝/降级模式

5. LangChain/LangGraph 经验：
   - 以状态机/图式编排代理流程
   - 中间件化注入：动态模型、动态工具、错误处理、观测埋点

## 核心范围（MVP -> V1）

- 对话与会话管理：多会话、流式输出、可中断/重试
- 工具系统：文件、Shell、Web、检索、MCP 接入
- 记忆系统：短期会话记忆 + 长期任务记忆（可检索）
- 开发代理：支持代码生成、修改、测试、提交建议
- 评测系统：任务成功率、工具调用成功率、回归基准
- 安全治理：策略拦截、审计日志、风险分级
- 运行平台：长时循环、失败恢复、健康检查、告警

## 里程碑

### M0（第 1 周）：可运行基线
- 打通当前 harness 初始化流程（`--bootstrap`）
- 打通持续循环（`--run-forever`）
- 产出首版 `feature_list.json` 与 `claude-progress.txt`

### M1（第 2~3 周）：核心 Agent 闭环
- 实现 ReAct 主循环
- 实现最小工具集（shell/file/web）
- 工具调用失败重试与错误分层

### M2（第 4~5 周）：安全与对齐
- 编写项目“行为宪法”与拒绝策略
- 增加高风险操作二次确认或沙箱执行
- 增加审计日志与可追踪 trace id

### M3（第 6~8 周）：Claude-like 体验
- 对话 UI、会话列表、消息流式渲染
- 工具调用可视化（步骤、参数、结果）
- 代码/工件渲染与下载

### M4（第 9~10 周）：无限自动化增强
- 失败自动恢复、断点续跑
- 任务队列与优先级调度
- 多代理协作（coder/tester/reviewer）

### M5（第 11~12 周）：生产化
- 压测、稳定性测试、成本优化
- 回归评测与发布门禁
- 部署 runbook 与运维告警完善

## 完成定义（DoD）

- `feature_list.json` 所有项 `passes: true`
- 关键链路 E2E 测试通过（对话、工具、代码修改、恢复）
- 稳定运行 72 小时无致命故障
- 高风险请求命中治理策略并有审计记录
- CI 全绿，核心性能与成本指标达标
